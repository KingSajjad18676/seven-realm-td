# LiveOps, Retention, Events, Shops

**Last updated:** 2026-06-04  
**Design canon:** [README_03_ETHICAL_MONETIZATION_BUSINESS.md](README_03_ETHICAL_MONETIZATION_BUSINESS.md) · [README_05_LAUNCH_LIVEOPS_COMMUNITY.md](README_05_LAUNCH_LIVEOPS_COMMUNITY.md)

Launch with a **polished authored game** and small repeatable modes — not a large battle-pass or subscription economy on day one.

---

## 1. Fair economy principles

- Core campaign playable without paying  
- No hard paywalls; no forced ads during battle  
- Cosmetics and supporter offerings are safest monetization  
- Paid options must not dominate power ([README_03](README_03_ETHICAL_MONETIZATION_BUSINESS.md) forbidden list)  
- Code may contain IAP/subscription **stubs** — treat as **post-launch / deferred**, not soft-launch catalog  

---

## 2. Launch monetization (when store opens)

| Product | Notes |
|---------|--------|
| Founder’s Supporter Pack | Ad removal + cosmetics + badge; no combat power |
| Permanent ad removal | One-time; disables optional rewarded prompts |
| Cosmetic bundles | Themed tower/hero/HUD/VFX skins — readable in combat |
| Authored expansion | After demand proven — full chapter, not cut content |

**Optional rewarded ads** (after core loop is fun): post-run bonus, Daily Tale reroll, collection lore preview, pre-run scout — never in combat, never required.

**Store tabs (calm):** Featured, Heroes, Towers, Effects, Profile, Supporter Pack, Restore Purchases.

---

## 3. Daily and weekly systems (post-launch cadence)

**Weekly:** validated Daily Tale seeds, one weekly challenge, bug triage, community post, support review.

**Monthly:** small cosmetic collection or curated bundle, community challenge, quality patch if needed, accessibility + balance review.

**Do not** promise weekly handcrafted maps unless the team can deliver ([README_05](README_05_LAUNCH_LIVEOPS_COMMUNITY.md)).

Legacy/scaffold systems in code (may exist but not launch-critical):

- daily login calendar, daily quests, rotating bazaar, hero dispatch  
- weekly boss trial, honor leaderboard, guild boss  

---

## 4. Events (playable without spending)

| Event | Systems | Rewards |
|-------|---------|---------|
| Nowruz Sacred Garden | Map modifiers + cosmetic tower set | Collection, profile frame |
| Damavand Echoes | Anchor-order variation, Endless modifiers | Banner, lore |
| Lion’s Oath | Khan 1 mastery | Rakhsh cosmetic |
| Manuscript Night | UI theme + accessibility-safe visibility | HUD cosmetic |
| Forge Festival | Hybrid-tower challenge | Forge tower skin |

Event structure when built: 7–14 days, fair modifier, cosmetic-forward rewards, optional event levels — avoid pay-only progression.

---

## 5. In-game shops (meta)

| Shop | Role |
|------|------|
| Daily Bazaar | Soft-currency rotation |
| Event Shop | Event currency cosmetics |
| Roguelite Merchant | Run-scoped picks |
| Cosmetic Shop | Direct cosmetic prices |
| Honor Shop | Cosmetic / honor currency |

Align honor and event currencies with README_03: limited currencies, clear sources/sinks.

---

## 6. Retention without manipulation

```
finish run → clear result → one meaningful reward → one new possibility → one-tap replay
```

Avoid: energy timers, streak-loss anxiety, forced store visits after defeat, confusing premium currency chains, launch battle pass pressure.

---

## 7. Analytics (live-ops relevant)

Track replay after win/defeat, Khan funnel, store views/conversion, rewarded-ad opt-in, fairness complaints, review sentiment — see [README_03](README_03_ETHICAL_MONETIZATION_BUSINESS.md) §9 and [README_05](README_05_LAUNCH_LIVEOPS_COMMUNITY.md) §6.

---

## 8. Live-ops stop rule

Reduce live-ops scope when it harms Khan 1 quality, crashes, or player trust. Fewer excellent events beat many rushed ones.
